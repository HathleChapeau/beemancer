################################################################################
#                        FEATURE: Magic Hive                                   #
################################################################################

Description: Ruche magique pour héberger les MagicBees avec gestion automatique
Statut: COMPLÈTE

================================================================================
ARCHITECTURE
================================================================================

- MagicHiveBlock: Bloc avec GUI
- MagicHiveBlockEntity: 12 slots (5 bee + 7 output), implements IHiveBeeHost + IHiveInternals
- HiveBeeLifecycleManager: Cycle de vie abeilles (release, entry, breeding, tick)
- IHiveBeeHost: Interface publique pour les entites abeilles (handleBeePing, onBeeKilled, etc.)
- IHiveInternals: Interface interne pour le lifecycle manager (getItems, shouldReleaseBee, etc.)
- HiveConfig: Parametres par type de ruche (MAGIC_HIVE: spawnHeight=1, inflate=2, entry=1.5)
- MagicHiveMenu: Slots custom (BeeAssignmentSlot, OutputOnlySlot)
- MagicHiveScreen: GUI avec layout honeycomb + indicateur breeding

================================================================================
SYSTÈME D'ASSIGNATION
================================================================================

- États: EMPTY → INSIDE → OUTSIDE → INSIDE (cycle)
- Assigner: Placer MagicBeeItem dans slot
- Abeille dehors: Slot devient "ghost", UUID trackée
- Retrait ghost: Abeille dehors despawn
- Mort abeille: Slot se libère
- Retour ruche: Bee capturée, état INSIDE, loot déposé

================================================================================
SÉCURITÉ UUID BEE-HIVE
================================================================================

- Bee ping ruche toutes les 2s (40 ticks) via tickHivePing() → handleBeePing()
- UUID mismatch + slot OUTSIDE → doublon détruit (verifyOutsideBees() lancé)
- UUID mismatch + slot INSIDE → bee adoptée, slot mis en OUTSIDE
- Ruche vérifie ses slots OUTSIDE toutes les 2s (serverTick timer)
- Entité absente → resetSlotToInside() avec cooldown récupération
- canBeeEnter() exige UUID match en plus de l'assignation
- releaseBee() reset la vélocité (setDeltaMovement(Vec3.ZERO))

================================================================================
DÉTECTION DE FLEURS
================================================================================

- Scan périodique (toutes les 5 secondes) des fleurs pour chaque abeille
- Utilise le FlowerGene de l'abeille et l'areaOfEffect de sa config
- Fournit une liste de fleurs à chaque abeille
- getAndAssignFlower(): Récupère et assigne une fleur à l'abeille
- returnFlower(): L'abeille retourne une fleur non visitée

================================================================================
DÉPÔT D'INVENTAIRE RÉCOLTEUSE
================================================================================

- Quand une récolteuse rentre, dépose tout son inventaire dans les slots output

================================================================================
SLOTS
================================================================================

- 0-4: BeeAssignmentSlot (5 emplacements abeilles)
- 5-11: OutputOnlySlot (7 slots honeycomb, extraction seulement)

================================================================================
INTERFACES ET DELEGATION
================================================================================

MagicHiveBlockEntity implements IHiveBeeHost + IHiveInternals.
Toute la logique bee lifecycle est deleguee a HiveBeeLifecycleManager:
- releaseBee, canBeeEnter, addBee, onBeeKilled, handleBeePing
- insertIntoOutputSlots, tickBeeSlot, checkReturningBees, verifyOutsideBees
- tryAutoBreeding (15% chance a l'entree si !antibreedingMode)

Semantique MagicHive:
- shouldReleaseBee(slot) = canBeeForage(slot) (fleurs, jour/nuit, temperature, pas antibreeding)
- shouldBreedOnEntry() = !antibreedingMode (crystal au-dessus = PAS de breeding)

Le lifecycle manager utilise HiveConfig.MAGIC_HIVE:
- spawnHeightOffset = 1 (bee spawn au-dessus du bloc)
- searchInflate = (2, 2, 2) (zone de detection bees retour)
- entryDistance = 1.5 (distance pour entrer dans la ruche)

MagicBeeEntity et ForagingBehaviorGoal utilisent instanceof IHiveBeeHost
(pas instanceof MagicHiveBlockEntity) pour interagir avec les deux types de ruches.
