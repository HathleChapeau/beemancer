Fais toi une equipe Forestry expert, Productive Bees expert, Create expert et un
expert de l'architecture de Beemancer.

  /home/adri/beemancer/beemancer-linux/bee_brainstorm.txt

== SUJET DU BRAINSTORM ==

Le comportement des abeilles et des ruches. Comment les ameliorer, les optimiser, et fix pour s'assurer d'avoir le comportement désiré.
On couvre:
- Le comportement IA des abeilles (goals, pathfinding, etats)
- Le cycle de vie dans les ruches (slots, lifecycle, foraging)
- La genetique et les especes (genes, breeding, mutations)
- Les nids naturels et la capture (worldgen nests, scoop)
- La progression des ruches (tiers, upgrades, capacites)
- L'interaction joueur-abeille (feedback visuel, sons, particules)
- Le gameplay loop global (pourquoi le joueur veut elever des abeilles)

- L'information doit être documenté et validé par les pairs
- Doit toujours faire l'objet de recherche sur internet, ainsi que de recherche complémentaire sur internet

   - Ce que chaque fichier fait

   - Les patterns utilises
   - Les interactions entre fichiers
   ** SANS jugement ni suggestion **

2. Les 3 experts proposent des ameliorations/nouvelles features basees sur
   leur mod de reference. Chaque proposition suit le format:
   [TAG] Titre proposition
     Quoi: Description
     Pourquoi: Motivation/probleme resolu
     Comment: Implementation technique dans le code Beemancer
     Reference: [MOD] Feature/classe/pattern de reference + source URL
     Priorite: P0 (quick win) a P4 (futur)
     Effort: Trivial/Faible/Moyen/Eleve
     Risque: Aucun/Faible/Moyen/Eleve

3. L'architecte challenge les propositions:
   - "Est-ce que ca existe deja dans le code?" (verifier)
   - "Quel fichier serait modifie?" (concret)
   - "Source?" (si pas de reference)
   ** TOUJOURS en posant des QUESTIONS, jamais en proposant **

4. Synthese avec convergences/divergences par le team-lead
