================================================================================
ARCHITECTURE — PHYSIQUE
================================================================================

Separation claire des responsabilites:

1. HoverbikeSettings (record immutable):
   maxHoverSpeed, maxRunSpeed, runThresholdSpeed,
   hoverAcceleration, runAcceleration, deceleration, brakeDeceleration, hoverFriction,
   gravity, terminalVelocity, rotationSpeedMax, rotationSpeedMin,
   gaugeFillRate, gaugeDrainRate, liftSpeed

2. HoverbikePhysics (classe stateless):
   - calculateHoverVelocity(): 4 directions, friction glace, gravite reduite
   - calculateRunVelocity(): avant uniquement, decel, brake, gravite
   - calculateYawDelta(): rotation progressive vers curseur joueur
   - shouldTransitionToRun() / shouldTransitionToHover(): conditions de mode
   - localToWorld() / worldToLocal(): conversion espaces

3. HoverbikeEntity (logique entite):
   - tickRidden(): rotation + transitions de mode + gauge fill
   - travel(): calcul velocite, envol, conversion world, inertie, collision feedback
   - move(): anti-tunneling (subdivision grandes vitesses, max 8 steps)
   - Edit mode: synched data, shader toggle, distance check

================================================================================
MODES DE DEPLACEMENT
================================================================================

Mode HOVER (par defaut):
- WASD: 4 directions relatives a l'orientation
- Acceleration: hoverAcceleration (0.008)
- Friction: hoverFriction (0.002) - style glace
- Gravite: reduite (0.025 vs 0.08 normal)
- Vitesse max: maxHoverSpeed (0.15)
- Rotation: constante (rotationSpeedMax 6 deg/tick)
- Envol: maintenir SPACE consomme jauge, monte a liftSpeed (0.06)

Mode RUN (sprint + seuil):
- W: accelere (runAcceleration 0.012)
- S: frein (brakeDeceleration 0.015)
- Rien: deceleration naturelle (0.003)
- Vitesse max: maxRunSpeed (0.6)
- Rotation: variable (6.0 → 1.5 deg/tick selon vitesse)
- FOV: boost +15 degres proportionnel a la vitesse

Transitions:
- HOVER → RUN: sprint + W + speed > runThresholdSpeed (0.1125)
- RUN → HOVER: speed < runThresholdSpeed (automatique)

================================================================================
SYSTEME DE PARTIES MODULAIRES
================================================================================

4 categories x 3 variantes = 12 pieces :
- Chaque variante a un modele 3D Java, une texture, un nom, et 2 stats
- Les variantes sont chargees depuis JSON (data-driven)
- Les modeles sont enregistres en Java (geometrie code)
- L'index de variante est synche via EntityDataAccessor (int par categorie)
- Changement de variante via HoverbikePartScreen → HoverbikeVariantPacket (C2S)

Categorie → Stats affectees:
  CHASSIS    : Hover_Max_Speed, Run_Max_Speed
  COEUR      : Hover_Acceleration, Run_Acceleration
  PROPULSEUR : Lift_Speed, Gauge_Fill_Rate
  RADIATEUR  : Hover_Friction, Brake_Deceleration

================================================================================
EDIT MODE
================================================================================

Activation: Shift+clic droit sur le hoverbike (toggle)
Effets:
  - Shader edit_darken.fsh assombrit tout sauf la moto (via masque entity_outline)
  - Les 4 parties s'ecartent avec animation smooth (AnimationController + MoveAnimation)
  - Panneau stats affiche en haut a gauche (HoverbikeEditStatsHud)
  - Raycast custom detecte la partie visee (HoverbikePartPicker → hitboxes locales)
  - Glow overlay blanc sur la partie hover
  - Clic droit sur partie → ouvre HoverbikePartScreen
  - Auto-exit si joueur trop loin (5 blocs)

================================================================================
ASSEMBLY TABLE
================================================================================

- Bloc slab (demi-bloc bas) pour exposer les pieces
- Accepte uniquement les HoverbikePartItem
- Clic droit avec piece = place. Clic droit main vide = retire
- Renderer: piece 3D rotative au-dessus du slab (AssemblyTableRenderer)
- Drop item on break

================================================================================
SYSTEME DE MODIFIERS (config JSON)
================================================================================

6 modifiers par defaut, 8 tiers chacun:
  Prefixes: Swift (speed), Turbocharged (accel), Agile (handling), Soaring (flight)
  Suffixes: of Stability (weight/aerodynamics), of Precision (handling/accel)

Chaque modifier contient:
  - name, is_prefix, tags[], T1_pool..T8_pool (poids random)
  - stat_objects[]: {statistic, value_type (+/%), T1..T8 ranges [min,max]}

Note: Les modifiers sont charges et disponibles mais pas encore appliques
      aux stats reelles de l'entite (systeme de roll/application a implementer).
