ENTITE PRINCIPALE — MagicBeeEntity.java (701L)
  - Extends Bee vanilla, FlyingMoveControl, noGravity
  - Attributs: HP=10, FLYING_SPEED=0.6, MOVEMENT_SPEED=0.3, ATTACK=2
  - 8 SynchedEntityData: species, environment, flower, pollinated, enraged,
    returning, debugDest, debugPath
  - Server fields: geneData, targetPos, homeNestPos, assignedHivePos/Slot,
    enragedTimer (200 ticks=10s), hivePingTimer (40 ticks)
  - tick(): force vol + tick enraged timer + tick hive ping + gene.applyBehavior()
  - NBT save/load: 66 lignes (27 save + 39 load)

5 GOALS ENREGISTRES:
  | Prio | Goal                          | Lignes | Description                    |
  |------|-------------------------------|--------|--------------------------------|
  | P0   | FloatGoal                     | vanilla| Flotte sur l'eau               |
  | P1   | ReturnToHiveWhenLowHealthGoal | 111L   | Fuite non-interruptible, v=0.12|
  | P2   | BeeRevengeGoal                | 143L   | Attaque, timer 10s, CD 20 ticks|
  | P3   | ForagingBehaviorGoal          | 500L   | Butinage complet, state machine|
  | P3   | WildBeePatrolGoal             | 361L   | Patrouille nids, 3 waypoints   |
  | P4   | MoveToTargetGoal              | 79L    | Navigation manuelle BeeWand    |

EXCLUSIVITE P3: ForagingGoal requiert hasAssignedHive(),
               WildBeePatrol requiert !hasAssignedHive()+hasHomeNest()

STATE MACHINE — BeeAIStateMachine.java (227L)
  6 etats: IDLE → LEAVING_HIVE → SEEKING_FLOWER → WORKING → RETURNING → RESTING
  Timeouts: SEEKING_FLOWER=800 ticks (40s), RETURNING=800 ticks
  ATTENTION: Utilisee SEULEMENT par ForagingBehaviorGoal.
  WildBeePatrolGoal a sa PROPRE state machine interne (PatrolState enum).

PATHFINDING — BeePathfinding.java (507L)
  Algorithme Theta* 3D (Any-Angle), 26 directions
  DDA 3D line-of-sight, wall penalty +0.5, altitude bonus -0.3
  Cache: invalidation si start > 3 blocs ou end change
  Replanning si hors chemin > 5 blocs, string pulling smoothing
  MAX_ITERATIONS=1000, MAX_PATH_LENGTH=100
  En pratique: recalcul toutes les ~25 ticks (1.25s) grace au cache

CYCLES DE COMPORTEMENT:
  Butinage: getAndAssignFlower → Theta* → approach from above → descend →
    work (pitch 40deg, 100 ticks) → setPollinated → return → enter hive
  Patrouille: sort nid → 3 waypoints aleatoires (rayon 15, 2-4 blocs au-dessus
    sol) → attend 2-10s chaque → retour nid → bee supprimee → respawn par nid
  Combat: hurt() → enraged 200 ticks → poursuit → attaque CD 20 ticks
  Fuite: vie < 30% → P1 non-interruptible → nav directe v=0.12

GENES EXISTANTS PERTINENTS (EnvironmentGene):
  - canWorkAtNight(): VERIFIE dans MagicHiveBlockEntity.canBeeForage() L382-389
  - canWorkInRain(): EXISTE mais N'EST VERIFIE NULLE PART
  - temperatureTolerance: VERIFIE dans canBeeForage(), echelle -2 a +2
  - 3 implementations: Normal(night=F, rain=F, tol=0), Nocturnal(night=T),
    Robust(rain=T, tol=1)
  - gene.applyBehavior() appele chaque tick mais TOUTES implementations sont NO-OP

POINT PERFORMANCE IDENTIFIE:
  FlowerSearchHelper.findAllFlowers() = scan brute-force O((2r+1)^3)
  Avec rayon=16: ~35,937 lookups de BlockState par appel
  C'est la cible #1 pour l'optimisation, impacte directement la recherche de
  fleurs, la pollinisation, et le territoire variable.

BUG POTENTIEL IDENTIFIE:
  Les deux state machines (ForagingGoal.BeeAIStateMachine vs
  WildBeePatrolGoal.PatrolState) ne communiquent PAS. Si une abeille sauvage
  est attaquee pendant la patrouille, BeeRevengeGoal prend le dessus (P2 vs P3)
  mais WildBeePatrolGoal n'est pas reset proprement.

DUPLICATION DE CODE IDENTIFIEE:
  navigateToVec3() duplique dans 5 fichiers avec variations de vitesse:
  | Goal           | Speed source          | Factor | Extra                    |
  |----------------|-----------------------|--------|--------------------------|
  | Foraging       | config.flyingSpeed    | * 0.2  | enraged check            |
  | WildPatrol     | config.flyingSpeed    | * 0.15 | zero distance guard      |
  | Revenge        | config.enragedSpeed   | / 10   | target entity center     |
  | FleeHP         | fixe                  | 0.12   |                          |
  | MoveTarget     | fixe                  | 0.08   |                          |
  La partie rotation (yaw+bodyRot) est IDENTIQUE dans les 5 cas.

================================================================================
1.4 PRINCIPE ARCHITECTURAL EMERGENT
================================================================================

Consensus entre les 3 experts sur une separation claire:

  BEHAVIOURS (pattern Create) = ACTIONS que l'abeille FAIT
    ForagingBehaviour, PatrolBehaviour, CombatBehaviour, EffectBehaviour...
    → Composables, encapsulent leur Goal + etat + NBT
    → Ajoutables/retirables sans modifier MagicBeeEntity

  GENES (pattern Forestry/PB) = MODIFICATEURS qui affectent les behaviours
    SpeedGene, TerritoryGene, TemperGene, ProductivityGene...
    → Heritables via breeding
    → Modifient les PARAMETRES des behaviours (pas les behaviours eux-memes)
    → Config JSON definit les defauts, le gene peut les depasser via breeding

  CONFIG JSON (existant) = DEFAUTS par espece
    foragingDuration, flyingSpeed, areaOfEffect, pollinationLoot...
    → Baseline definies par le modder/datapack
    → Les genes MODIFIENT ces valeurs, ne les remplacent pas
