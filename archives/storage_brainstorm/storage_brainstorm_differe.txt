[D] Grid Services / Behaviours composables
---------------------------------------------------------------------------
Proposé par: Expert AE2
Catégorie: Amélioration
Priorité: P4

Description:
  Refactorer les services du controller en "Grid Services" composables,
  similaires à l'architecture d'Applied Energistics 2.
  Chaque service (stockage, livraison, crafting) serait un module
  indépendant qui peut être ajouté/retiré dynamiquement du réseau.

Référence:
  AE2: IGridService, ITickingService, ICraftingService
  Create: ProcessingBehaviours (contraptions modulaires)

Motivation:
  - Séparation claire des responsabilités
  - Facilite l'ajout de nouveaux services (futur crafting, énergie, etc.)
  - Testabilité améliorée (services isolés)

Prérequis:
  - Compléter les changements Groupe 1 (extraction managers, IItemHandler)
  - Analyser l'architecture Create pour les Behaviours composables
  - Définir l'interface IGridService avec lifecycle (attach/detach/tick)

Notes:
  L'utilisateur souhaite étudier la version Create (Behaviours) plutôt que
  la version pure AE2 (Grid Services). À évaluer après stabilisation.

[AD] Serial numbers
---------------------------------------------------------------------------
Proposé par: Expert AE2
Catégorie: Amélioration
Priorité: P3

Description:
  Ajouter un numéro de série (compteur monotone) au cache incrémental
  pour détecter si le cache est stale (par exemple après un crash/reload).
  Chaque modification du cache incrémente le serial. Les snapshots incluent
  le serial, permettant de détecter les divergences client/serveur.

Référence:
  AE2: IGridStorage serial, MEInventoryHandler version tracking

Motivation:
  - Détection robuste de cache stale après crash
  - Synchronisation optimiste client/serveur
  - Fallback automatique vers full scan si serial mismatch

Prérequis:
  - Cache incrémental [U] implémenté (fait dans Groupe 2)
  - ViewerSyncManager opérationnel (fait dans Groupe 2)
  - Évaluer la fréquence réelle de cache stale en production

Notes:
  Le cache incrémental a un full scan backup toutes les ~100 ticks qui
  devrait couvrir la plupart des cas. Les serial numbers ajouteraient
  une couche de sécurité supplémentaire pour les edge cases.
