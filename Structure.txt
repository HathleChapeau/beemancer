--------------------------------------------------------------------------------
FEATURE: Bee Behavior System
--------------------------------------------------------------------------------
Description: Système de comportement automatique des abeilles (butinage, combat, fuite)
Statut: COMPLÈTE

Architecture:
- BeeBehaviorType: Enum FORAGER/HARVESTER
- BeeBehaviorConfig: Configuration par espèce (vitesse, vie, loot, etc.)
- BeeBehaviorManager: Chargement depuis JSON, lazy loading
- BeeActivityState: États de la machine à états (IDLE, SEEKING_FLOWER, WORKING, RETURNING)
- Goals AI: ReturnToHiveWhenLowHealthGoal, BeeRevengeGoal, ForagingBehaviorGoal, HarvestingBehaviorGoal

Cycle de vie butineuse:
1. Abeille sort de la ruche (après cooldown)
2. Cherche une fleur du type défini par FlowerGene
3. Se pose sur la fleur et travaille (foragingDuration ticks)
4. Devient pollinisée (pollinated=true)
5. Retourne à la ruche
6. Dépose le loot dans les slots output
7. Cooldown aléatoire (restCooldownMin à restCooldownMax)
8. Régénération si blessée (cooldown ne démarre qu'à vie pleine)

Combat:
- Si attaquée → devient enragée (vitesse augmentée)
- Attaque l'agresseur (BeeRevengeGoal)
- Si vie < 30% → fuite immédiate vers ruche (ReturnToHiveWhenLowHealthGoal)

Paramètres configurables (par espèce):
- Communs: flyingSpeed, enragedFlyingSpeed, health, attackDamage, regenerationRate
- Agressivité: aggressiveToPlayers, aggressiveToPassiveMobs, aggressiveToHostileMobs
- Cooldown: restCooldownMin, restCooldownMax
- Butineuse: foragingDuration
- Récolteuse: harvestingDuration, inventorySize
- Loot: pollinationLoot (item, minQty, maxQty, chance)

Nouveaux FlowerGenes:
- trees: Arbres (logs + leaves)
- mushrooms: Champignons
- crystals: Améthyste et cristaux
- nether_wart: Verrues du Nether

Config: data/beemancer/behavior/species_behaviors.json
Tags: data/beemancer/tags/blocks/bee_flowers_*.json
--------------------------------------------------------------------------------

FEATURE: Gene System
--------------------------------------------------------------------------------
Description: Système modulaire de gènes pour les abeilles
Statut: COMPLÈTE

Architecture:
- GeneCategory: Catégorie de gènes (species, environment, flower, lifetime)
- Gene: Interface définissant un gène
- AbstractGene: Implémentation de base
- GeneRegistry: Registre central
- BeeGeneData: Stockage des gènes + lifetime d'une abeille

Catégories de gènes:
1. SPECIES (Espèce) - Couleur: Or
   - common: Standard
   - noble: +production, -vitesse
   - diligent: +vitesse, -production

2. ENVIRONMENT (Environnement) - Couleur: Vert
   - normal: Conditions standard
   - robust: Travaille sous la pluie
   - nocturnal: Travaille la nuit

3. FLOWER (Fleur) - Couleur: Violet
   - flowers: Fleurs standard
   - crops: Cultures
   - cave_plants: Plantes de caverne
   - trees: Arbres (logs + feuilles)
   - mushrooms: Champignons
   - crystals: Améthyste
   - nether_wart: Verrues du Nether

4. LIFETIME (Durée de vie) - Couleur: Aqua
   - short: ~5 minutes
   - normal: ~20 minutes
   - long: ~60 minutes

FEATURE: Magic Hive
--------------------------------------------------------------------------------
Description: Ruche magique pour héberger les MagicBees avec gestion automatique
Statut: COMPLÈTE

Architecture:
- MagicHiveBlock: Bloc avec GUI
- MagicHiveBlockEntity: 12 slots (5 bee + 7 output), tracking états, breeding, cooldowns
- MagicHiveMenu: Slots custom (BeeAssignmentSlot, OutputOnlySlot)
- MagicHiveScreen: GUI avec layout honeycomb + indicateur breeding

Système d'assignation:
- États: EMPTY → INSIDE → OUTSIDE → INSIDE (cycle)
- Assigner: Placer MagicBeeItem dans slot
- Abeille dehors: Slot devient "ghost", UUID trackée
- Retrait ghost: Abeille dehors despawn
- Mort abeille: Slot se libère
- Retour ruche: Bee capturée, état INSIDE, loot déposé

Nouveautés:
- Cooldowns par slot (beeCooldowns[])
- Régénération automatique des abeilles blessées (beeNeedsHealing[])
- Dépôt automatique du loot de pollinisation
- Auto-release quand cooldown terminé et vie pleine

Slots:
- 0-4: BeeAssignmentSlot (5 emplacements abeilles)
- 5-11: OutputOnlySlot (7 slots honeycomb, extraction seulement)

--------------------------------------------------------------------------------
FEATURE: Breeding System
--------------------------------------------------------------------------------
Description: Système de reproduction des abeilles avec cristal et incubateur
Statut: COMPLÈTE

Architecture:
- MagicBreedingCrystalBlock: Active le mode breeding quand au-dessus d'une ruche
- BreedingManager: Logique de breeding et d'héritage des gènes
- BeeLarvaItem: Item larve avec gènes hérités
- IncubatorBlock/BlockEntity: Transforme larve en abeille

Fonctionnement:
1. Placer cristal au-dessus de la ruche → mode breeding activé
2. Minimum 2 abeilles INSIDE + 1 slot output vide
3. Tentative de breeding: 5% par seconde
4. Une abeille perd 10% de son lifetime max par tentative
5. Résolution espèce: species_combinations.json
6. Si "nothing" → pas de larve créée
7. Sinon → BeeLarvaItem créée avec gènes hérités

Règles d'héritage:
- Parents espèces différentes → traits par défaut pour offspring
- Même espèce mais résultat différent → traits par défaut
- Même espèce ET même résultat → chaque trait hérité aléatoirement d'un parent

Incubation:
- Slot unique acceptant BeeLarvaItem
- Timer: 600 ticks (30 secondes)
- Transforme larve en MagicBeeItem avec mêmes gènes

Config: data/beemancer/breeding/species_combinations.json

8. GUIDE D'INTÉGRATION D'UNE FEATURE
================================================================================

Pour ajouter un nouveau gène:
1. Créer une classe dans content/gene/[category]/
2. Hériter de la classe de base de la catégorie (ex: SpeciesGene)
3. Enregistrer dans GeneInit.registerAllGenes()
4. Ajouter la config dans data/beemancer/genes/[category].json
5. Ajouter les traductions dans lang/*.json

Pour ajouter un nouveau FlowerGene:
1. Créer une classe dans content/gene/flower/
2. Hériter de FlowerGene
3. Définir le TagKey<Block> pour les blocs cibles
4. Enregistrer dans GeneInit.registerAllGenes()
5. Créer le tag dans data/beemancer/tags/blocks/
6. Ajouter les traductions

Pour ajouter une nouvelle catégorie de gènes:
1. Ajouter dans GeneCategory.java
2. Créer le package content/gene/[category]/
3. Créer la classe de base [Category]Gene
4. Ajouter la config JSON
5. Ajouter les traductions

Pour ajouter une nouvelle combinaison de breeding:
1. Modifier data/beemancer/breeding/species_combinations.json
2. Ajouter un bloc { "parent1": "...", "parent2": "...", "results": [...] }
3. Les probabilités doivent totaliser 100%
4. "nothing" = pas de descendant

Pour configurer une nouvelle espèce:
1. Modifier data/beemancer/behavior/species_behaviors.json
2. Ajouter un bloc dans "species" avec l'ID de l'espèce
3. Définir behaviorType (FORAGER ou HARVESTER)
4. Configurer les paramètres (vitesse, vie, loot, etc.)
