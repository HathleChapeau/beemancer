================================================================================
FEATURE: Crafter System (Crafting Automatique)
================================================================================

Description: Systeme de crafting automatique connecte au reseau de stockage.
Le joueur programme des recettes (vanilla 3x3 ou machine) sur des Crafting
Papers, les charge dans le Crafter, et les abeilles livrent les ingredients
puis ramenent les resultats.

Statut: IMPLEMENTE

================================================================================
ARCHITECTURE
================================================================================

Le systeme se compose de 6 couches:

1. DATA: CraftingPaperData, PartCraftingPaperData, CraftTask
2. ITEMS: CraftingPaperItem, PartCraftingPaperItem
3. BLOCK: CrafterBlock, CrafterBlockEntity, CraftManager
4. MENU/GUI: CrafterMenu, CrafterScreen, CrafterMachinePanel
5. NETWORK: CrafterGhostSlotPacket, CrafterInscribePacket, CrafterModeTogglePacket
6. INTEGRATION: Delivery system (DeliveryTask craft fields), Import/Export craft mode

================================================================================
FONCTIONNEMENT
================================================================================

MODE CRAFT (recettes vanilla 3x3):
1. Le joueur ouvre le Crafter et place des Crafting Papers vierges dans le slot reserve
2. Il clique sur les ghost slots avec des items pour definir la recette 3x3
3. Le preview affiche le resultat si la recette est valide
4. Clic Inscribe: consomme 1 paper vierge, cree 1 Crafting Paper inscrit dans output A
5. Le paper inscrit va dans la bibliotheque (8 slots)

MODE MACHINE (recettes non-vanilla):
1. Toggle CRAFT/MACHINE via le bouton
2. Ajout de lignes input (+) avec item ghost + quantite
3. Definition de l'output avec item ghost + quantite
4. Inscribe: consomme 1 paper vierge, cree 2 Part Crafting Papers (INPUT + OUTPUT) avec UUID partage

CRAFT AUTOMATIQUE:
1. Le CraftManager recoit une demande de craft via submitCraft()
2. Resolution recursive des sous-crafts (bibliotheque, max profondeur 8, detection cycles)
3. Les abeilles apportent les ingredients au crafter (DeliveryTask -> IDeliveryEndpoint)
4. CraftManager.tick() verifie si tous les ingredients sont presents dans le craftBuffer
5. Execute le craft, resultat retourne au reseau via abeille preloaded (bypassCapacity)
6. Timeout 2 minutes: annule et retourne les items orphelins au reseau

LINKING CONTROLLER:
1. Shift+Right-Click sur le Controller pour entrer en edit mode
2. Right-Click sur le Crafter pour lier (1 crafter max par controller)
3. Le crafter est controller-only (pas de relay)

IMPORT/EXPORT CRAFT MODE:
1. Toggle [C] sur l'interface import/export
2. Placer un Part Crafting Paper INPUT dans l'import interface
3. Placer un Part Crafting Paper OUTPUT dans l'export interface
4. Les interfaces routent via le controller commun vers le crafter lie

================================================================================
DEPENDANCES
================================================================================

- StorageControllerBlockEntity (link crafter, delivery management)
- StorageNetworkRegistry (NetworkBlockType.CRAFTER)
- StorageDeliveryManager (craft delivery tasks)
- DeliveryBeeEntity + DeliveryPhaseGoal (transport ingredients/resultats)
- IDeliveryEndpoint (reception items par le crafter)
- InterfaceTask pattern (modele pour CraftTask)
- GuiRenderHelper (rendu programmatique screens)
- AbstractApicaScreen (base class mais CrafterScreen extend directement AbstractContainerScreen)

================================================================================
CONTRAINTES DESIGN
================================================================================

- 1 crafter = 1 controller maximum (pas de relay)
- 1 controller = 1 crafter maximum
- 1 seul craft a la fois dans le crafter
- Detection de cycles dans les crafts imbriques (Set visited + profondeur max 8)
- CraftBuffer cap: 54 stacks max
- Timeout craft: 2400 ticks (2 minutes)
- Retour resultat TOUJOURS via abeille (pas de depot direct)
